# Design

## Purpose

This document defines the user experience and visual design principles for Factor.

Design decisions should support the product mission:

> Help users discover meaningful relationships between factors in their lives.

---

## Core Principles

### Understanding Over Tracking

Factor is not a tracking application.

Data collection exists to support understanding.

The interface should prioritize insights, patterns, and relationships over data entry.

---

### Simplicity Over Features

Every screen should focus on a single purpose.

Avoid unnecessary controls, configuration, and visual clutter.

---

### Signal Over Noise

Important information should stand out naturally.

Emphasize:

* insights
* trends
* anomalies
* relationships

De-emphasize raw data whenever possible.

---

### Progressive Disclosure

Reveal complexity gradually.

Keep common workflows simple.

Advanced analytical capabilities should appear only when relevant.

---

### Calm Interface

The application should feel focused and reflective.

Avoid visual noise, excessive animation, and attention-seeking elements.

---

## Visual Principles

### Minimalism

Use whitespace, typography, and layout as primary design tools.

Decorative elements should be minimized.

---

### Consistency

Similar concepts should look and behave consistently throughout the application.

---

### Meaningful Color

Color should communicate information rather than decoration.

---

## Success Criteria

A successful design helps users:

* capture observations quickly
* identify meaningful patterns
* understand relationships between factors
* make better decisions from evidence
